# AMD fine-tune — verifiable evidence

Screenshots are easy to fake; these two files aren't. Both are raw artifacts
pulled directly out of the running system, not summaries or write-ups.

## 1. `codeverse_theme_sft.jsonl` — the real distillation dataset

569 training examples (545 train / 24 held-out), each one a real chat
transcript generated by the teacher model (Fireworks `glm-5p2`) from
CodeVerse's actual production prompts, and validated by the app's own
parsers before being kept. This is the exact file that was uploaded into the
AMD Instinct notebook and used for the LoRA run — see
`codeverse_finetune_amd.ipynb` in this folder for the training code that
consumes it.

```
wc -l amd/codeverse_theme_sft.jsonl   # 569
```

## 2. `evidence_amd_generated_theme.json` — a database row, not a claim

A raw export of a `theme_dictionaries` row from CodeVerse's **production
Postgres database**, pulled with:

```sql
SELECT row_to_json(t) FROM (
  SELECT id, theme_name, llm_provider, llm_model, created_at, mappings, rationale
  FROM theme_dictionaries WHERE llm_model = 'codeverse-student'
) t;
```

Key fields:

| Field | Value |
|---|---|
| `theme_name` | `"Chess"` |
| `llm_provider` | `"openai_compatible"` — the generic client CodeVerse uses to talk to any OpenAI-compatible endpoint |
| `llm_model` | `"codeverse-student"` — the served name of our LoRA-tuned Gemma 4, reachable only through the AMD Instinct reverse tunnel |
| `created_at` | `2026-07-12T14:21:20.98317+00:00` |
| `mappings` | all 210 Python concepts, mapped by the AMD-hosted model |

This row could only exist if a real HTTP request left the production
backend, crossed the reverse SSH tunnel, reached the FastAPI server running
on the AMD Instinct GPU, and got a real completion back — the row is written
*after* that round trip succeeds, inside the same `/themes/generate`
transaction described in this repo's `backend/src/codeverse_api/routers/themes.py`.

Independent verification: query the live database directly, or re-run the
same chip flow on the deployed app whenever the AMD tunnel is up (chip
clicks route to AMD by design; typed text always uses the production
provider — see `themes.py` for the routing and automatic fallback).
